# S&S Automations — Design System

The component library for **S&S Automations LLC**, built from the `ss-brand` skill. It encodes the brand's locked palette, type scale, and named "signature moves" as real, renderable components so any design — in Claude Design or in code — comes out on-brand.

> Every business leaks. We find where your business leaks time and money — then we plug it.

## Layout

```
styles.css            Single entry stylesheet (@imports tokens + brand.css)
tokens/               colors.css · typography.css · spacing.css (CSS variables, source of truth)
brand.css             The class vocabulary every surface is built from
assets/               Logo PNGs (black-on-light, off-white-on-dark, both transparent)
components/
  foundations/        Colors · Typography · Logo · EmDashLabel
  print/              HeroPanel · CloserPanel · WarmGrayStrip · SectionBlock · TwoColumnSplit · BusinessCard
  web/                CtaButtons · WebSection · MarketplaceTile · Storefront · ProductDetail
  document/           DocumentPage
_preview/             Verification screenshots (generated by verify.mjs)
styleguide.html       One page that renders the whole system
verify.mjs            Renders every preview in headless Chromium → _preview/<Name>.png
```

Each component is a directory of four files:
- `<Name>.html` — standalone preview card (first line is the `@dsCard` marker for the Claude Design pane)
- `<Name>.jsx` — the React component the design agent codes with
- `<Name>.d.ts` — the props contract
- `<Name>.prompt.md` — usage reference (real class names, rules, a snippet)

## The one rule that governs everything: context

S&S has **three contexts** and the layout discipline differs by context (palette/voice/type are identical across all three):

| Context | Proportion | Hero/Closer panels? | Output |
|---|---|---|---|
| **Print / Card** | ~70% off-white / 25% black / 5% warm gray | Yes | PDF, PPTX |
| **Web / Long-form** | ~90% off-white / 10% black | **No** — flat off-white, black only for CTAs | HTML |
| **Document Mode** | ~100% off-white | **No** | DOCX / clean PDF |

The "open with a black hero, close with a black panel" reflex is correct in **Print** and wrong on the **Web**.

## Locked invariants (never violate)
- Colors only from the palette. Never `#ffffff` (use `#f5f2ed`), never `#000000` (use `#0a0a0a`). No accent colors.
- No gradients, drop shadows, glows, or rounded corners. Hard rectangular edges everywhere.
- Left-align everything (only the warm-gray strip centers).
- Every headline ends with a period. Heavy neo-grotesque type only.

```bash
node verify.mjs        # renders all previews → _preview/, fails on any JS error
```
